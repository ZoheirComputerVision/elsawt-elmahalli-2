# دستور المشروع — الصوت المحلي | The Local Echo

اهتمام محلي ... التزام وطني

---

## 1. الرؤية

بناء أول منصة إعلامية محلية رقمية في الجزائر تغطي الدوائر والبلديات بعمق ميداني، مع نموذج قابل للتوسع وطنياً.

## 2. الرسالة

تمكين المواطن المحلي من الوصول إلى أخبار وخدمات ومعلومات موثوقة تخص محيطه المباشر، مع الحفاظ على أعلى معايير المهنية والشفافية.

## 3. المبادئ التوجيهية

- **الموثوقية**: جميع الأخبار تخضع للتدقيق قبل النشر
- **الجهوية**: التركيز على الشأن المحلي بغطاء وطني
- **الشمولية**: كل دائرة وكل بلدية تمثل وحدة تغطية مستقلة
- **الشفافية**: سجل تدقيق كامل لكل عملية تحرير ونشر
- **التوسع المنهجي**: كل ولاية جديدة تضاف دون إعادة تصميم النظام

## 4. هيكل التوسع الاستراتيجي

| المرحلة | الإطار الزمني | النطاق | الأهداف |
|---------|---------------|--------|---------|
| 0: التجهيز | الأسبوع 1–2 | تيارت (المقر) | بناء MVP، إعداد البنية التقنية، عزل البيانات جغرافياً، تأسيس Wilaya → Daira → Commune |
| 1: الشق الأول | الشهر 1–3 | تيارت · السوقر · قصر الشلالة | إطلاق المنصة فعلياً، تأسيس غرف أخبار محلية، بناء شبكة مراسلين، اختبار النموذج الاقتصادي |
| 2: التغطية الكاملة | الشهر 4–6 | جميع دوائر ولاية تيارت | تغطية شاملة للولاية، مراسل في كل دائرة، دليل اقتصادي شامل، خدمات محلية |
| 3: التوسع التاريخي | الشهر 7–9 | ولاية تيسمسيلت | تكرار نموذج تيارت، بناء شبكة مراسلين جديدة، إطلاق نسخة جهوية متكاملة |
| 4: التوسع الغربي | الشهر 10–12 | ولاية سعيدة | توسيع الحضور الجهوي، تعزيز القاعدة الإعلانية، استقطاب المؤسسات الاقتصادية |
| 5: الهضاب والجنوب | السنة 2 | الأغواط · البيض · النعامة | بناء شبكة إعلامية جهوية مترابطة، توسيع التغطية الصحراوية والرعوية |
| 6: الامتياز الوطني | السنة 3 | وطني | فتح ولايات جديدة تدريجياً، توحيد البنية التقنية، تشغيل منصة متعددة الولايات |

## 5. النموذج الجغرافي

```
Wilaya (ولاية)
  └── Daira (دائرة)
        └── Commune (بلدية)
              └── Correspondent (مراسل)
                    └── News / Directory / Ads
```

## 6. النموذج التقني

| العنصر | القيمة |
|--------|--------|
| Framework | Next.js 16 (App Router) |
| اللغة | TypeScript (strict) |
| قاعدة البيانات | PostgreSQL |
| ORM | Prisma |
| المصادقة | Auth.js (NextAuth v5) |
| التخزين | محلي → Supabase Storage (قريباً) |
| الواجهات | TailwindCSS + shadcn/ui |
| الاستضافة | HostingGuru.io |

## 7. اتفاقيات التسمية

- **المجلدات**: `kebab-case` (مثال: `economic-directory`)
- **المكونات**: `PascalCase` (مثال: `NewsService`)
- **المسارات**: `kebab-case` (مثال: `/api/economic-directory`)
- **قواعد البيانات**: `snake_case` (مثال: `featured_stories`)

## 8. مبادئ الحدود

- لا يتم تعديل الصفحة الرئيسية إلا ضمن خطة معتمدة
- لا يتم تعديل سير العمل التحريري بعد الاستقرار
- لا يتم تعديل محرك البحث الأساسي بعد الاستقرار
- كل ميزة جديدة تبدأ بوثيقة تصميم في `/features/`

## 9. مسار اتخاذ القرارات

```
اقتراح → وثيقة في /features/
  → مراجعة → موافقة → تنفيذ
  → اختبار → نشر → توثيق
```
